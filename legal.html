2. Privacy Policy

FEP Compass is a 100% client-side application. There is no backend server. No part of this app transmits your data to any server operated by the developer, because no such server exists.

All application state — your AI provider configuration, FEP limit trackers, declarations, chat session history and activity log — is stored exclusively in your browser's localStorage, under the keys fep_cfg, fep_limits, fep_decls, fep_sess and fep_activity. This data never leaves your device except where you explicitly configure an AI provider (see §3 below).

There is no analytics, telemetry, tracking pixel or third-party advertising script anywhere in this codebase. Nothing about your usage of the app is collected, logged, or reported to the developer.

If you configure a Gemini API key, that key is stored only in your browser's localStorage and is sent only directly from your browser to Google's Gemini API endpoint when you make a request — never to any developer-operated server. You are responsible for safeguarding any device or browser profile where you save an API key.

This policy is written with Malaysia's Personal Data Protection Act 2010 (PDPA) in mind: because the app processes data entirely on-device and does not collect, store, or transmit personal data to the developer, the developer does not act as a "data user" under the PDPA in respect of your use of this app. Any personal or transaction data you type into the app remains under your own control on your own device, and (where applicable) is subject to your own organisation's data-handling policies.

3. Third-Party AI Provider Disclosure

The AI Advisor and Compliance Analyst features require you to connect a third-party AI provider. No first-party server is in the loop at any point — requests go directly from your browser to the provider you choose:

  • Gemini (cloud): if you configure a Gemini API key, your query text — including any text extracted from documents or images via OCR/PDF reading, and your chat history for that session — is sent directly from your browser to generativelanguage.googleapis.com, Google's own API endpoint. This is governed by Google's own terms and privacy policy, not the developer's.
  • Ollama (local): if you select Ollama, requests are sent only to the local or loopback URL you specify (by default http://localhost:11434), i.e. to a model running on your own machine or network. Nothing is sent to the internet in this mode.

You should independently review the relevant provider's own privacy and data-handling terms before sending any sensitive or confidential transaction details through either path.

4. Data Retention

Because there is no backend, there is no server-side data retention of any kind — the developer cannot retain data that is never transmitted to it.

Locally, data persists in your browser's localStorage until you clear it yourself, either via Settings → "Clear all local data," or by clearing your browser's site storage/cache for this app. Chat session history is automatically capped at 30 stored entries, and the activity/audit log is automatically capped at 50 entries (MAX_ACTIVITY) — older entries are dropped as new ones are added.
